# 1. 분석 과제 발굴 개요
## 1. 분석 과제 발굴의 개념과 탐색 방법
### 1. 분석 과제 발굴
- 무엇을 분석해야 하는지 그 분석 과제를 발굴하는 일
- 데이터 분석 문제로 변환 하는 것을 포함하는 개념
- 정의서 형태로 도출
- 상향식과 하향식으로 나뉨
### 2. 분석 과제 탐색 방법
- 하향식 접근법
    - 정통적인 Top-Down 수행 방법으로 단계화되어 문제 해결
- 상향식 접근법
    - 데이터의 조합속에 인사이트를 찾아내는 Botton-Up 방식
#### 개념도
- 분석 대상이 무엇인지 알고 있는가
    - Yes
        - 하향식 접근법
            - 1. 문제 탐색 단계
                - 비즈니스 모델 탐색 기법
                - 분석기회 발굴 및 범위 확장
                - 외부 참조 모델 기반 문제 탐색
                - 분석 유스케이스
            - 2. 문제 정의 단계
                - 식별된 비즈니스 문제를 데이터 문제로 변환하여 과제를 정의
            - 3. 해결 방안 탐색 단계
                - 과제 정의 후 어떻게 해결한 것인지 방안을 탐색
            - 4. 타당성 검토 단계
                - 경제적 타당성, 기술적 타당성등을 검토
    - No
        - 상향식 접근법
            - 1. 지도/비지도 학습
                - 지도 학습
                - 비지도 학습
            - 2. 프로토타입
                - 시행착오 해결법

# 2. 하향식 방법
## 1. 1단계 문제 탐색 단계
### 1. 비즈니스 모델 탐색 기법
- 업무(Operation)
- 제품(Product)
- 고객(Customers)
- 문제를 발굴하고 규제와 검사, 지원 인프라의 두가지 영역에 대한 기회를 추가로 도출하는 작업
- 인력을 영역을 의미
### 2. 분석 기회 발굴 범위의 확장
- 거시적 관점
    - 문제 혹은 변화가 기업에 주는 영향을 탐색
    - 사회
    - 기술
    - 경제
    - 환경
    - 정치
- 경쟁자 확대 관점
    - 기업에 위협이 될 상황을 탐색
    - 대체재
    - 경쟁자
    - 신구 진입자
- 시장의 니즈 탐색
    - 시장의 니즈 탐색 관점에서 문제를 탐색
    - 고객
    - 채널
    - 영향자들
- 역량의 재해석
    - 역량의 재해석 관점에서 다시 기업 내부를 확인
    - 내부 역량
    - 파트너와 네트워크
### 3. 외부 참조 모델 기반 문제 탐색 및 분석 유스케이스 정의
- 시사점을 도출
- Pool을 통한 Quick&Easy 분석 기회가 무엇인지 아이디어를 얻는 브레인토핑
- 분석 유스케이스 정의
- 유스케이스: 분석을 적용했을 때 업무 흐름을 개념적으로 설명
## 2단계 - 문제 정의 단계
### 1. 문제 정의
- 식별된 비즈니스 문제를 데이터의 문제로 변환하여 정의하는 단계
- What,Why 수행 하는 관점이라면 -> 필요 데이터 및 기법을 정의하는데 위한 데이터 분석 문제로 변환
### 2. 문제 정의 예
- 고객의 불만이 높다 -> 불만의 요인을 분석하고 상관관계 및 예측 모델 수립
## 3단계 - 해결 방안 탐색 단계
### 1. 해결 방안 탐색
- 정의된 데이터 분석 문제를 해결하기 위한 방안을 모색하는 단계
- 가능한 시스템을 갖추었는지 분석을 수행할 인력이 확보 확인
### 2. 해결 단계 프로세스
- 기존 시스템으로 가능
    - 분석 역량이 있음 -> 기존 시스템과 인력으로 수행
    - 분석 역량이 없음 -> 역량을 확보(교육/채용)
- 기존 시스템으로 불가능
    - 분석 역량이 있따 -> 시스템 고도화
    - 분석 역량이 없다 -> 전문업체 아웃소싱
## 4단계 - 타당성 검토 단계
### 1. 타당성 검토
- 해결 방안이 타당한 것인지 검토
- 경제적 타당성 검토와 데이터 및 기술적 타당성 검토
### 2. 타당성 검토의 두 가지 유형
- 경제적 타당성
    - 비용이 분석을 수행했을때 경제적 이득보다 크면 기업이 손해임
- 데이터 및 기술적 타당성
    - 경제적 측면이 아무리 뛰어나도 분석 역량을 고려해야함
#### 사례로 하향식 접근법
- 항공사 고객 이탈 사례
- 1단계 - 문제 탐색 단계
    - 비즈니스 모델 기반 문제 탐색
    - 외부 사례 기반 문제 탐색
- 2단계 - 문제 정의 단계
    - 1 단계의 문제를 기반으로 문제 분석
- 3단계 - 해결 탐색 단계
    - 2단계를 기반으로 분석 역량, 시스템 등의 해법을 찾음
- 4단계 - 타당성 검토
    - 3단계를 기반으로 경제적, 기술적 요건 확인하여 검토
# 3. 상향식 접근
## 1. 상향식 접근법 개요
### 1. 상향식 접근 개념
- 원천 데이터을 통해 통찰과 지식을 얻는 접근 방식
### 2. 상향식 접근의 특징
- 먼저 분석부터 시작하고 결과로부터 가치를 찾음
#### 예시
- 빅데이터 -> 분석 -> 가치
## 2. 지도학습과 비지도학습
### 1. 지도학습
- 정답이 있는 데이터를 활용하여 데이터 분석
- 레이블이 범주형과 연속형으로 나눔
### 2. 비지도 학습
- 정답이 없는 데이터로 학습
- 군집화하여 새로운 데이터를 예측
- 필드 값을 구하는 것이 아닌, 데이터 결합,연관성,유사성을 상태를 표현
## 3. 시행착오를 통한 문제 해결법(프로토타이핑 접근법) 
### 1. 개념
- 상향식 접근법 중 하나로 시행착오 해결법
- 하향식이 없는경우 하나의 방법으로 좋은 대안이 될 수 있음
- 문제 정의가 불확실성과 신규라면 프로토타이핑이 유용할 수 있음
### 2. 프로세스
- 1. 가설생성
- 2. 디자인에 대한 실험
- 3. 실제 환경에서의 테스트
- 4. 테스트 결과로부터 인사이트를 도출 및 확인
### 3.프로토타이핑 접근법이 필요한 경우
- 문제에 대한 인식 수준이 낮거나 불명확할 경우
- 필요 데이터의 존재 여부가 불확실할 경우
- 데이터의 사용 목적이 고정되지 않고 변화할 경우
